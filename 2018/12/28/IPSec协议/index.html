IPSec概述

安全服务

  • 保密性
  • 数据完整性
  • 访问控制
  • 数据源认证
    根据安全策略（针对安全需求给出的一系列解决方案，它决定了对什么样的通信实施安全保护以及何种安全保护）对IP数据报进行安全处理（加密，消息认证码，重新封装）

处理过程

协商阶段

  • 互联网密钥交换协议（IKE）
  • 通信双方互相认证对方身份，并根据安全策略协商使用的加密，认证算法，生成共享的会话密钥

    数据交互阶段

  • 认证首部（AH）
  • 封装安全载荷（ESP）
  • 通信双方利用协商好的算法和密钥对数据进行安全处理

    IPSec外出数据处理流程


    外出数据：应用IPSec,查询SAD
    IPSec引擎根据安全策略（SP）和安全关联（SA）调用密码算法对IP报文做安全处理（报文封装）

IPSec数据封装

AH（Authentication Header）

AH提供的安全服务：

  • 数据完整性（Integrity）
  • 数据源发认证（Data Origin authentication）
  • 抗重放攻击（Anti-replay attack）

    AH特点：

  • 不对传输的数据提供保密性
  • 密码学运算开销小
  • 两种工作模式（传输模式，隧道模式）

ESP（Encapsulating Security Payload）

ESP提供的安全服务：

  • 保密性
  • 数据完整性
  • 数据源发认证
  • 抗重放攻击

    报文格式

    字段说明：

  • SPI：协商SA时确定的安全参数索引
  • 序号：报文编号，用于防止重放攻击
  • 载荷：加密数据，与应用模式相关
  • 填充：1.加密算法采用分组加密时，明文数据需要填充。2.数据在一个4-byte边界对齐
  • 认证数据：变长字段。可选字段，Integrity Check Value(ICV),认证算法必须说明ICV的长度，比较规则和验证的处理步骤

    发送数据处理过程

  1. 查询SAD，获取安全参数
  2. 报文加密
  3. 生成序号
  4. 计算认证数据

    接收数据处理过程

  6. 根据<目的ip地址，安全协议，SPI>三元组查询SAD，若找不到有效SA，则丢弃报文
  7. 使用滑动窗口机制验证序号，防止重放攻击
  8. 验证认证数据，如果验证失败，则丢弃该报文

IKE：身份认证方法

主模式：使用数字签名认证方法

  • 1，2报文完成SA协商
  • 3，4报文传输密钥交换信息和随机数，完成密钥交换
  • 5，6报文进行身份和认证信息交换。

    主模式：使用公钥加密认证方法


    1，2报文完成SA协商
    3，4报文进行密钥交换和用对端公钥加密的身份信息及随机数
    5，6报文中可选的HASH（1）用于说明所使用的回应方公钥，其输入为所选公钥对应的证书
